Reverse Engineering a Suspicious PDF File

Published on May 24, 2026 • Written by NetOps Admin

PDF files are ubiquitous in corporate environments, making them a prime target for threat actors. In this post, we will dissect a malicious PDF that uses embedded JavaScript to execute a payload.

1. The Initial Assessment

First, we never open a suspicious file on a host machine. We move it to an isolated sandbox environment. Using a tool like pdfid, we scan for embedded elements.

$ pdfid suspicious_invoice.pdf
 PDF Header: %PDF-1.4
 obj           : 15
 endobj        : 15
 stream        : 4
 endstream     : 4
 /JS           : 2
 /JavaScript   : 2
